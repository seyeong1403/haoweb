# SANGGONG_REFERENCE_AUDIT
> 마스터 레퍼런스: https://sanggong.co.kr/ · 라이브 직접 접속 분석 (PC 1280–1440 / Mobile 390)
> 목적: 상공의 **구성 원리·화면 리듬·모션 문법**을 분석해 하오웹 콘텐츠·브랜드(레드 유지)로 재해석. 문구·로고·이미지·코드는 복사하지 않음.
> 분석일: 2026-07-15 · 확인 방식: DOM/computed style 측정 + 스크롤·리사이즈·요소 검사. (이 환경은 스크린샷·IO 프레임이 얼려 정지 캡처는 미첨부, 구조·수치는 실측)

---

## 1. 메인페이지 전체 섹션 순서
페이지 길이 ≈ 21스크린(≈15,300px). 히어로(다크/영상) → 이후 **흰 배경 카테고리 쇼케이스**의 반복.

1. **Hero** (`section-fullheight main_visual`) — 100svh, 풀블리드 영상 슬라이더(영상 3개) + 좌측 대형 텍스트 + 우측 작업 롤링 쇼케이스
2. **Rolling band** (`section_rolling`) — 큰 텍스트가 가로로 흐르는 마퀴 밴드(구축 유형·수상 등)
3. **Responsive Homepage** (`horizontal-scroll`) — 반응형 홈페이지 구축 사례 (가로 스크롤 갤러리)
4. **Shopping Mall** — 쇼핑몰 제작 사례
5. **Mobile Web & App** (`section-arc-images`) — 모바일웹/앱 사례 (아치형 이미지 배열)
6. **Design** — 포트폴리오·CI 브랜딩·편집디자인 사례
7. **Landing Promotion** (`section-pricing-cards`) — 랜딩/프로모션 사례
8. **Web Program & Solution** (`section-portfolio-grid`) — 웹프로그램·솔루션 사례
9. **Footer**

> 핵심: **포트폴리오(작업물) 중심** 구조. 각 카테고리 = "영문 대형 라벨 + 국문 부제 + 사례 갤러리 + All Works 더보기".

---

## 2. 각 섹션의 레이아웃 구조
- **Hero**: 좌(텍스트: 카테고리·대형 제목·설명·CTA·슬라이드 컨트롤) / 배경(풀블리드 영상). 우측 상단으로 작업명이 큰 크림색 글자로 롤링.
- **카테고리 쇼케이스**: 상단 큰 헤더(영문+국문) → 프로젝트 카드 갤러리(가로 스크롤 / 그리드 / 아치). 각 카드: **대형 이미지 + 위에 프로젝트명(대형 흰색) + View Project + 한 줄 설명**. 하단 "All Works 더보기".
- 콘텐츠 섹션 배경 = **흰색**. 히어로만 다크/영상. 섹션 사이 여백 큼(≈ clamp 100–150px).

---

## 3. 타이포그래피 위계 (실측)
- 폰트: **Poppins**(영문 디스플레이) + **NanumSquareNeo**(국문 본문). 
- **Hero 헤드라인: 124px / weight 200(가늘게)** — "크게+가늘게"가 시그니처(굵게가 기본인 룩과 반대).
- **Hero 작업 롤링: 90px / weight 800 / 크림색(#FDF9CF)**.
- **섹션 헤더(영문+국문): 70px / weight 800 / near-black(#0C0C0D)** — 영문 라벨과 국문 부제를 같은 큰 크기로.
- **프로젝트명(카드): 54px / weight 900 / 흰색** (이미지 위), split-text로 등장.
- **오버레이 메뉴: 80px / weight 100(아주 얇게)**.
> 위계 = 아주 큰 대비. 얇은 초대형 ↔ 굵은 대형을 교차. 자간 과도하지 않음.

---

## 4. 이미지 비율·크롭
- Hero 영상: object-fit **cover**, 풀블리드(화면 가득). 텍스트는 그 위 다크 오버레이.
- 프로젝트 이미지: 대형(한 카드가 화면 상당 부분). 라운드 모서리 사용. 일부 화면 밖으로 크롭·가로 스크롤.
- 아치형(Mobile 섹션) 등 섹션마다 이미지 배열 방식이 다름 → **화면마다 구성이 달라짐**(같은 카드 반복 아님).

---

## 5. 주요 모션 (라이브 확인)
엔진: **Lenis(스무스 스크롤) + GSAP + ScrollTrigger + Swiper**.
- **Hero 슬라이드(3)**: Swiper. 배경 영상/작업이 전환, 텍스트는 마스크 안 교체, 작업명 큰 글자 롤링. `.slider-arrow` prev/next + 자동 전환.
- **스크롤 리빌**: split-text(제목이 글자/단어 단위로 마스크에서 올라옴). 무의미한 전체 Fade-up 반복 아님 — 제목 위주.
- **가로 스크롤 갤러리**(Responsive Homepage 등): 세로 스크롤을 가로 이동으로.
- **앰비언트**: 영상 미세 확대/드리프트.
- 속도·easing 통일, 명확하게 끝남. 3D/글리치/네온/파티클 없음.

---

## 6. Hover 인터랙션
- 헤더 링크 hover: 색 변화(절제).
- 프로젝트/서비스: 이미지·텍스트 반응(hover 시 강조·전환). 카드가 위아래로 튀는 식의 남용 없음.
- 버튼: 배경/테두리 상태 변화(발광 없음).

---

## 7. Header · 전체 메뉴 · 페이지 전환
- **Header: fixed·투명·100px**. 좌측 로고(≈174px, left 30px). 우측: 영문 네비(AI SERVICE / PORTFOLIO / ABOUT US / BUSINESS / STORY / HIRING / CONTACT US) + **"menu" 버거** + **제작문의 CTA**.
- **버거 클릭 → `.menu-overlay`**: 풀스크린, **초대형 얇은(80px, weight 100) 영문 메뉴어** 세로 나열. 현재 페이지 표시·연락 CTA. (본문 스크롤 잠금 추정)
- 스크롤 후 헤더 가독성 확보 상태로 전환.

---

## 8. 모바일 변화 (390px)
- 헤더: **제작문의 + 대표번호 + 버거**만 노출(영문 네비는 오버레이로). 
- Hero: ≈ 뷰포트 높이(675px), 텍스트/영상 유지.
- 가로 오버플로 없음. 대형 타이포가 화면 안에 맞춰 축소.
- 오버레이 메뉴로 전체 항목 접근.

---

## 9. 하오웹에 **적용할** 요소 (재해석)
- **포트폴리오/작업물 중심** 페이지 골격(카테고리별 대형 사례 갤러리).
- **크게+가늘게 헤드라인**(초대형 얇은 국문) ↔ 굵은 대형 교차의 **강한 타이포 대비**.
- **영문 라벨 + 국문 큰 부제** 섹션 헤더(단, 남용 금지 — 브리프 20).
- **풀스크린 오버레이 메뉴**(초대형 얇은 메뉴어, 순차 등장, 스크롤 잠금, ESC/포커스).
- **Hero: 좌 텍스트(번호·카테고리·제목·설명·CTA·prev/next) / 우 대형 비주얼 프레임**(라운드·크롭), 슬라이드 3, 마스크 텍스트 교체 + 시간차.
- **화면마다 달라지는 레이아웃**(가로 스크롤·비대칭·아치 등), **절제된 모션**(split 리빌·마스크·앰비언트), Lenis+GSAP.
- 흰 콘텐츠 섹션 + 다크 히어로/CTA의 배경 대비 리듬.

## 10. **복사하지 않고 재해석할** 요소 (금지)
- 상공 고유 **문구·로고·이미지·프로젝트명·아이콘·코드** — 절대 복사 금지.
- **컬러: 민트(#29D4C1)+크림(#FDF9CF) 복사 금지** → 하오웹 **Black·Off-white·White·HAOWEB Red**로. 레드는 포인트만(슬라이드 번호/링크hover/키워드/인디케이터/Primary CTA/카테고리/메뉴 활성).
- 폰트: Poppins+NanumSquareNeo 그대로 쓰지 말고 하오웹 톤(Pretendard 본문 + 개성 있는 헤드라인)으로.
- 상공의 5개+ 카테고리 전부가 아니라 **하오웹 콘텐츠 구조**(Statement·Why·Brand Integration·Process·Search&AI·Diagnosis 등)에 맞춰 재구성.
- 가짜 프로젝트/성과 금지 → 실제 자료 전까지 **정확한 비율의 중립 레이아웃 프레임 + TODO 주석**(AI 합성 사무실 사진으로 채우지 않음).
