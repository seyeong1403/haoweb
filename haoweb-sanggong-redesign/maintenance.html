AREAS

영역별로
무엇을 다루나요.

유지관리는 여러 성격의 작업을 함께 다룹니다. 각 영역이 어디까지 포함되고, 어떤 경우 별도 확인이 필요한지 먼저 설명합니다. 구체적인 상품·대응 시간·요금은 상담에서 안내합니다.

텍스트·이미지 수정
이미 있는 페이지의 문구·사진·정보를 교체합니다. 대체로 유지관리 범위에서 진행합니다. 다만 페이지 구조 자체를 바꾸는 개편은 별도로 확인합니다.
제품·서비스 추가
기존과 같은 형식으로 새 제품·서비스 항목을 등록합니다. 형식이 같으면 유지관리 범위이고, 새 화면 구조나 기능이 필요하면 별도로 협의합니다.
포트폴리오·사례 추가
새 프로젝트와 사례를 기존 갤러리 형식에 맞춰 등록합니다. 이미지·설명이 준비되어 있으면 바로 반영합니다.
FAQ·칼럼 추가
질문과 칼럼 같은 축적형 콘텐츠를 등록합니다. 원고가 준비되어 있으면 등록만 진행하고, 원고 작성이 필요하면 콘텐츠 제작 범위로 함께 협의합니다.
기능 수정
게시판·문의 폼·연동 등 기존 기능의 작은 보완과 조정을 다룹니다. 새로운 기능을 처음부터 개발하는 일은 별도 범위입니다.
오류 대응
표시가 깨지거나 동작이 멈추는 문제를 점검하고 수정합니다. 원인에 따라 대응 방식과 소요가 달라져, 확인 후 안내합니다.
보안·백업
정기 백업과 기본적인 보안 점검으로 안정적인 운영을 돕습니다. 서버 이전이나 대규모 인프라 변경은 별도로 협의합니다.
분석 및 개선
방문과 문의로 이어지는 흐름을 살펴 개선점을 제안합니다. 분석 도구를 새로 도입하거나 전면 개편하는 일은 별도 범위입니다.
별도 범위 확인이 필요한 작업
새 기능 개발, 대규모 개편, 촬영·브랜딩 등은 유지관리와 성격이 다릅니다. 필요할 때 범위와 방식을 별도로 확인해 진행합니다.

모든 작업을 하나의 상품으로 묶기보다, 필요한 영역만 골라 진행하는 편이 효율적입니다. 어떤 영역이 자주 필요한지는 홈페이지의 성격과 운영 방식에 따라 다르므로, 실제 범위는 상담에서 함께 정리합니다. 축적형 콘텐츠가 많은 홈페이지라면 콘텐츠 추가가, 기능이 많은 홈페이지라면 기능 점검이 더 자주 필요합니다.

SCOPE

포함 범위와
별도 확인의 구분.

같은 요청이라도 규모에 따라 처리 방식이 달라집니다. 아래는 일반적인 구분 기준이며, 실제 포함 범위는 상담에서 확정합니다.

작업대체로 유지관리 범위별도 확인·협의
콘텐츠 수정문구·이미지·정보 교체페이지 구조 자체를 바꾸는 개편
항목 추가기존 형식의 제품·사례·FAQ 등록새 화면·기능이 필요한 추가
기능기존 기능의 작은 보완·오류 수정새 기능 개발·연동 신설
운영·보안백업·보안 점검·안정성 관리서버 이전·대규모 인프라 변경
개선흐름 점검과 개선 제안분석 도구 도입·전면 리뉴얼

FLOW

요청은
이렇게 진행됩니다.

수정이나 추가가 필요할 때, 요청을 받은 뒤 범위를 확인하고 반영해 함께 점검하는 흐름으로 진행합니다. 급한 오류와 일반 수정은 처리 방식이 다를 수 있어, 상황에 맞게 안내합니다.

  • 요청 접수 — 바꿀 내용과 위치를 한곳에 모아 전달받습니다
  • 범위 확인 — 유지관리 범위인지, 별도 협의가 필요한지 구분합니다
  • 작업 진행 — 확인된 범위에서 수정·추가를 반영합니다
  • 함께 점검 — 반영 결과를 확인하고 마무리합니다
  • 기록 정리 — 무엇을 바꿨는지 정리해 다음 운영에 참고합니다

구체적인 유지관리 상품과 대응 시간, 요금은 상담에서 안내합니다. 홈페이지 상태와 필요한 범위에 따라 달라지므로, 현재 상황과 원하는 운영 방식을 알려 주시면 맞는 방식을 함께 정합니다.

SCOPE

유지관리 범위.

아래는 큰 틀의 구분입니다. 구체적인 포함 범위와 대응 시간, 요금은 홈페이지 상태와 필요에 따라 달라져 상담에서 확정해 안내드립니다.

01

포함되는 작업

정기 콘텐츠 수정, 기존 형식의 항목 추가, 오류 대응, 정기 백업과 기본 보안 점검처럼 운영을 이어 가기 위한 작업입니다.

02

별도 견적이 필요한 작업

새 기능 개발, 대규모 개편, 촬영·브랜딩, 서버 이전처럼 성격과 규모가 다른 작업은 별도로 범위를 확인해 진행합니다.
