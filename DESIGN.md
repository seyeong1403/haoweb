# 하오웹 — 디자인 틀 (Design Framework)

브랜드의 시각 기준 문서입니다. 색·타이포·간격의 **단일 소스**는 `css/tokens.css`이며,
`css/style.css`는 이 토큰을 참조합니다.
(최종 갱신 2026-07-14 — 영상 Hero + 카피 전환, 차별점 3축, 세부페이지 콘텐츠 보강 반영)

---

## 0. 브랜드 정체성 (가장 중요)

하오웹은 **AI 진단·SEO 컨설팅을 파는 회사가 아니라, 고객의 홈페이지·앱을 기획·디자인·개발하는 제작 브랜드**입니다.
SEO·AEO·GEO·AI 검색 대응은 **정체성이 아니라 홈페이지를 제작할 때 적용하는 설계 역량** — WEB&APP 역량·프로세스·FAQ 안에서만 설명합니다.

**차별점 = 결합된 제작 방식 (두 축).** 한 문장: *“업종을 이해하고, 브랜드의 모든 접점을 한 팀이 연결합니다.”*
- **축1 · Customer Journey**: `검색 → 비교 → 신뢰 → 문의` — 업종별 고객의 탐색·문의 흐름을 설계.
- **축2 · Brand Integration**: `BRANDING → CONTENT & PHOTO → EDITORIAL & CATALOG → WEB & APP → GROWTH` — 브랜딩·촬영·편집디자인·카탈로그·웹·앱을 하나의 기준으로 연결·확장.

메인 차별점 섹션(`#strength .difference`)은 **핵심 3축을 먼저 제시**(2026-07-14 재구성, `.axes3`):
- **01 CUSTOMER JOURNEY** `검색 → 비교 → 신뢰 → 문의`
- **02 BRAND INTEGRATION** `브랜딩 → 콘텐츠·촬영 → 편집·카탈로그 → 웹·앱`
- **03 GROWTH STRUCTURE** `서비스 → 사례 → FAQ → 칼럼 → 검색·AI → 운영`
동일 카드가 아니라 큰 번호 + 라벨 + 흐름(연결선) + 짧은 설명 + 스크롤 활성화(`.axis3.on`). 그 아래 **5가지 제작 원칙**(`.bi-steps`, 3~4줄 축약)을 연결하고, 마무리 문장(`.bi-close`, “화면만 디자인하지 않습니다…”)으로 닫음. 자세한 내용은 각 서비스 페이지로.

## 2. 컬러 시스템

- **방향**: 오프화이트/밝은 배경을 전체의 60~70%로. **다크 섹션은 Hero·포트폴리오·최종 CTA 등 2~3곳만.**
  (전체를 어둡게 하지 않음 — AI 기술회사처럼 보이지 않도록.)
- 3-레이어 토큰(Primitive → Semantic → Component)은 `css/tokens.css`가 단일 소스.
- 레드(`--color-primary #E83817`)는 **CTA·핵심 문장·강조에만** 절제 사용. 넓은 면은 중립색(잉크/페이퍼).

## 5. 시그니처 컴포넌트 (메인, 2026-07-13 재설계)

- **Hero — 영상 배경 + 카피 전환 (2026-07-14)**: 두 개의 `<video>` 레이어(`heroVidA/B`)를 겹쳐 **크로스페이드(1.2s)**로 순환. 영상과 카피를 하나의 데이터 배열 `heroSlides`(`video/eyebrow/title/lead`)로 관리 — 영상이 바뀌면 라벨·제목·설명도 함께 전환(문구 페이드아웃 → 영상 교차 → 중반 560ms에 새 문구 교체 → 제목→설명→CTA 순 등장). `01 / 02` 진행표시(`.hero-idx`). 제목 `min-height: 3줄` 예약으로 줄 수가 달라도 CTA 위치 안정. `prefers-reduced-motion`=첫 슬라이드·포스터 정적. 예비 슬라이드는 `heroSlides` 주석으로 준비(미노출). 슬라이드1 라벨 `AI SEARCH READY`, 슬라이드2 `BRAND INTEGRATION`.
- **브랜드 통합 4단계 (.bi-grid / .bi-steps)**: 좌측 타이틀 고정(sticky) + 우측 4단계(01 브랜드 / 02 콘텐츠 / 03 웹·앱 / 04 확장) 세로 배치. 스크롤에 따라 현재 단계 활성화(.on) + 좌측 연결선(.bi-rail-fill) 진행. 마무리 한 문장(.bi-close). 카드·비교박스·그래디언트 없음 — 타이포·번호·라인만.
- **서비스 인터랙티브 (.svc-ix)**: 좌측 서비스 목록(번호+이름) + 우측 선택 서비스 상세(설명·제작 대상·핵심 전략·상세 링크). hover·click 전환(데스크톱), 모바일=세로 리스트. `js/main.js` 3b-3.
- **포트폴리오 (.port-feat + .port-alt)**: 대표 1개 크게 + 이후 좌우 교차(.pa-flip). 실제 자료 전까지 와이어프레임 플레이스홀더('실제 프로젝트 이미지 입력 예정'). 가짜 회사·수치 없음.
- **고객 인터뷰 (.intv)**: 하나의 큰 인터뷰 구조 — 큰 인용 / 프로젝트·고객 / 문제·해결·결과 / 상세보기. 인용문·이름은 자료 확보 후(플레이스홀더).
- **제작 프로세스 (.proc-grid)**: 좌측 현재 단계 대형 번호+제목(고정) + 우측 7단계 설명(01 상담 / 02 기획 / 03 콘텐츠(브랜딩·촬영·원고·카탈로그) / 04 디자인 / 05 개발 / 06 검수 / 07 오픈·운영). 스크롤로 단계 전환 + 진행선(.proc-track-fill). `js/main.js`.
- **세부페이지 메인 톤 이식 (2026-07-13)**: 7개 세부페이지도 메인 느낌 반영 — ① 페이지 히어로 대형화(.page-hero-title, 메인 스케일) ② 프로세스 `.steps`를 세로 번호 레일 + 스크롤 활성화(7열 격자 폐기, 다크 섹션) ③ 브랜드 체인(.brand-chain)을 세로 번호 스텝으로(색은 `inherit`/`currentColor`로 다크·라이트 양쪽 대응) ④ 섹션 배경 오프화이트(#E8E8ED)/순백(#F9F9FB paper)/다크 교차.
- **브랜드 체인(.brand-chain)** · **여정(.pjourney)**: 세부페이지의 업종별 흐름(예: 환자 탐색, 구매 동선, 상담 절차).
- **페이지 내 내비(.page-nav)**: 긴 세부페이지 상단 sticky 이동.
- **정직한 문의 CTA(.cta-actions)**: 실제 폼 전송 연결 전에는 가짜 '신청 완료' 없이 전화·이메일 안내 + 개인정보처리방침 링크.
- 스크롤 딤 효과(.bi-step / .proc-item)는 `body.gsap-on` + 데스크톱에서만 적용 → GSAP 미로드·모바일 시 전체 표시(콘텐츠 안 사라짐).

### 5-1. 세부페이지 구조 — 공통 섹션 vs 업종별 고유 섹션 (2026-07-14)

**공통 섹션(모든 제작서비스 페이지 동일 틀):** `page-hero`(도입 문구 `lead-intro`) → `page-nav`(sticky 인페이지 이동) → 방문자 탐색 과정(`.jrny`) → HAOWEB METHOD(`.method`, 5관점) → 제작 프로세스(`.steps`) → 제작 범위·비용·기간 기준(`.spec-grid`) → 제작 사례/인터뷰(`port-ph` 플레이스홀더, 가짜 없음) → 관련 랜딩페이지(`.rel-links` → `landing.html#앵커`) → FAQ(`.faq-list`, 8~10+) → 관련 칼럼 → 최종 CTA.

**업종별 고유 섹션(페이지마다 다른 실제 정보):** 기업=자료→웹 재구성(`.deflist`)·정보구조(`.infostruct`) / 병원=병원 유형별(`.cmp-table`)·환자 탐색·의료광고 확인 원칙 / 변호사=전문분야 구조·상담 절차·법률광고 확인·사례 비식별 원칙 / 쇼핑몰=몰 종류(`.cmp-table`)·구매 여정·상세페이지 기획 / 프랜차이즈=업종별 차이·창업 절차·가맹 문의 / 앱=앱 종류·네이티브/하이브리드/웹앱 선택 기준(`.cmp-table`)·제작 흐름 / 랜딩=일반 홈피와 차이(`.cmp-table`)·유형·전환 흐름·업종별 앵커 허브(`#corporate-campaign`·`#hospital-consult`·`#legal-consult`·`#commerce-promo`·`#franchise-recruit`·`#app-launch`).

**신규 공통 컴포넌트(style.css 하단):** `.jrny`(가로 여정, 모바일 세로) · `.infostruct`(정보구조 태그) · `.checklist`(2열 체크) · `.deflist`(자료→웹 dt/dd) · `.cmp-wrap>.cmp-table`(비교표, 가로스크롤 보호) · `.method`(HAOWEB METHOD 번호+질문+답) · `.rel-links>.rel-link`(관련 랜딩/콘텐츠) · `.lead-intro`·`.prose`(본문 리듬). 반복 카드 대신 표·여정·프로세스로 교체하기 위한 것.

**각 서비스 페이지 차별점 노출:** 페이지에 바로 들어와도 (1) 하오웹이 그 업종을 어떤 관점으로 이해하는지 (2) 일반 화면 제작과 무엇이 다른지 (3) 홈페이지 외 브랜딩·촬영·카탈로그·웹·앱을 어떻게 연결하는지가 HAOWEB METHOD + 업종 고유 섹션에서 드러남. 공통 문구 복붙 금지, 업종 실제 정보로 작성.

## 6. 데이터 정직성 원칙 (엄수)

- 가상 회사명·성과 수치·고객 후기·인용문을 **공개 화면에 넣지 않음**. 자료 없으면 플레이스홀더(‘등록 예정’).
- 미확정 문구는 화면에 단정하지 않고 **HTML 주석 `TODO`** 로만 표시(‘확인 필요’ 텍스트 노출 금지).
- 결과 보장 표현 금지. 의료·법률은 광고 규정 유의.
- 필요한 실제 자료 목록: `자료-요청-목록.md`.
