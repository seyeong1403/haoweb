# 하오웹 — 디자인 틀 (Design Framework)

브랜드의 시각 기준 문서입니다. 색·타이포·간격의 **단일 소스**는 `css/tokens.css`이며,
`css/style.css`는 이 토큰을 참조합니다.
(최종 갱신 2026-07-09 — 실제 구현과 일치하도록 정리)

---

## 0. 브랜드 정체성 (가장 중요)

하오웹은 **AI/SEO 컨설팅 회사가 아니라, 브랜드 통합 제작 브랜드**입니다.
브랜딩·콘텐츠·촬영·편집디자인·카탈로그·웹·앱을 **하나의 브랜드 기준**으로 연결합니다.
AI 검색·SEO·AEO·GEO는 **헤드라인 상품이 아니라 제작 역량 중 하나** — WEB&APP 역량·프로세스·FAQ 안에서만 설명합니다.

핵심 차별점 = **통합 제작**: `BRAND STRATEGY → CONTENT PRODUCTION → WEB & APP → BRAND EXPANSION`
(동일 카드 나열이 아니라 연결 구조·타이포 중심으로 표현. 분리 제작 vs 통합 제작을 비방 없이 구조 차이로 비교.)

## 2. 컬러 시스템

- **방향**: 오프화이트/밝은 배경을 전체의 60~70%로. **다크 섹션은 Hero·포트폴리오·최종 CTA 등 2~3곳만.**
  (전체를 어둡게 하지 않음 — AI 기술회사처럼 보이지 않도록.)
- 3-레이어 토큰(Primitive → Semantic → Component)은 `css/tokens.css`가 단일 소스.
- 레드(`--color-primary #E83817`)는 **CTA·핵심 문장·강조에만** 절제 사용. 넓은 면은 중립색(잉크/페이퍼).

---

## 3. 타이포그래피 (실제 로드 폰트)

| 역할 | 폰트 | 용도 |
|------|------|------|
| 디스플레이·본문 | **Pretendard** | 제목/본문. 굵기·크기 대비로 위계 |
| 라벨·숫자·유틸리티 | **Space Grotesk** (`--font-label`) | eyebrow·단계 번호·영문 라벨·캡션 |

> ⚠️ 두 폰트만 사용(과거 문서의 'JetBrains Mono'는 실제 미사용 — 삭제함).
> **한글 가독성 최우선**: 제목에 `word-break: keep-all` 필수. 제목을 글자 단위로 흩지 않음. Hero 문장은 2~3줄에 바로 읽히게.

---

## 4. 레이아웃 원칙

- 섹션마다 **배경만 바꾸지 말고 레이아웃 자체를 다르게**: 초대형 타이포 / 좌우 분할 / 세로 목록 / 비교 구조 / 연결 흐름(체인) / 여정(단계).
- 카드 반복을 줄이고 타이포·연결 구조 중심.
- 반응형: 가로 스크롤 없음. 주요 브레이크포인트 900/720/560/390.

---

## 5. 시그니처 컴포넌트

- **브랜드 통합 시스템(.bs-flow)**: BRAND→CONTENT→WEB&APP→EXPANSION을 연결선·화살표로 이은 흐름. (구 'AI 답변 시뮬레이션 카드'는 폐기 — Hero는 이미지 없이 타이포 중심.)
- **비교(.compare / .bs-compare)**: 일반 제작 vs 하오웹 통합 제작, 하오웹 열 강조.
- **브랜드 체인(.brand-chain)** · **여정(.pjourney)**: 세부페이지의 업종별 흐름(예: 환자 탐색, 구매 동선, 상담 절차).
- **페이지 내 내비(.page-nav)**: 긴 세부페이지 상단 sticky 이동.
- **정직한 문의 CTA(.cta-actions)**: 실제 폼 전송 연결 전에는 가짜 '신청 완료' 없이 전화·이메일 안내 + 개인정보처리방침 링크.
- **아이콘**: `.ic` 인라인 스트로크(통일: stroke 1.7·round) + 서비스 구분용 DB 아이콘(`assets/icons/`, mask). 모든 제목 옆에 넣지 않고 선별 사용.

---

## 6. 데이터 정직성 원칙 (엄수)

- 가상 회사명·성과 수치·고객 후기·인용문을 **공개 화면에 넣지 않음**. 자료 없으면 플레이스홀더(‘등록 예정’).
- 미확정 문구는 화면에 단정하지 않고 **HTML 주석 `TODO`** 로만 표시(‘확인 필요’ 텍스트 노출 금지).
- 결과 보장 표현 금지. 의료·법률은 광고 규정 유의.
- 필요한 실제 자료 목록: `자료-요청-목록.md`.

## 파일 구조
```
haoweb/
├─ index.html           ← 메인
├─ company.html         ← 기업     ├─ hospital.html   ← 병원
├─ lawyer.html          ← 변호사   ├─ shop.html       ← 쇼핑몰
├─ franchise.html       ← 프랜차이즈 ├─ landing.html   ← 랜딩
├─ app.html             ← 앱
├─ css/tokens.css       ← 토큰(단일 소스)   ├─ css/style.css ← 스타일
├─ js/main.js           ├─ assets/(logo, icons)
├─ 자료-요청-목록.md    ← 관리자용 실제 자료 요청
└─ DESIGN.md            ← 이 문서
```
> 색/폰트/간격은 **`css/tokens.css`** 에서 관리.
> ※ style.css는 버전 블록(v2~v24)이 누적돼 있어 **공통 컴포넌트 기준 재구조화가 예정된 과제**(#16).
