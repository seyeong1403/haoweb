GEO — 정보의 관계

생성형 AI는 낱개의 페이지가 아니라 정보의 관계를 봅니다. 이 회사가 무엇을 하고, 어떤 서비스가 있고, 무슨 근거가 있는지가 서로 연결되어 있어야 정확하게 이해됩니다.

실제 작업

GEO에서 하오웹이 하는 일

정보 관계 설계
회사 → 서비스 → 프로젝트 사례 → 전문 콘텐츠 → 근거 정보가 연결되는 구조를 설계합니다.
일관된 표현
회사명·서비스명·주소 등 기본 정보를 전 페이지에서 동일하게 표기합니다.
구조화 데이터
화면의 콘텐츠와 일치하는 범위에서 스키마(구조화 데이터)를 적용합니다. 화면에 없는 정보를 넣지 않습니다.
근거 연결
주장(잘합니다)이 아니라 근거(과정·사례·기준)로 말하고, 각 주장에서 근거로 링크합니다.

구조 예시

연결된 정보는 이렇게 생겼습니다.

흩어진 정보

회사소개 따로, 서비스 따로, 블로그 따로. 서로 링크가 없고 표현도 제각각이라 AI가 같은 회사의 이야기인지 파악하기 어렵습니다.

연결된 정보

서비스 페이지가 관련 사례를 링크하고, 사례가 사용한 과정을 링크하고, FAQ가 서비스를 링크합니다. 어디서 시작해도 회사의 전체 그림에 도달합니다.

점검 항목

우리 홈페이지 GEO 체크

  • 회사·서비스·사례·콘텐츠가 서로 링크로 연결되는가
  • 회사명·서비스명이 전 페이지에서 동일하게 표기되는가
  • 주장마다 근거(과정·기준·사례)가 연결되는가
  • 구조화 데이터가 화면 내용과 일치하는가

AI가 반드시 활용한다고 보장하지 않습니다. 이해되기 쉬운 관계를 갖추는 것까지가 제작의 역할입니다.

자주 묻는 질문

GEO에 대한 질문

SEO와 뭐가 다른가요?
SEO는 페이지가 발견되는 구조, GEO는 발견된 정보들이 서로 연결되어 이해되는 구조입니다. 페이지 하나가 아니라 사이트 전체의 관계를 다룹니다.
구조화 데이터만 넣으면 되나요?
아닙니다. 스키마는 보조 수단이고, 핵심은 실제 콘텐츠가 연결되어 있는가입니다. 화면에 없는 정보를 스키마로만 넣는 것은 오히려 문제가 됩니다.
