# CORE_PAGE_MOTION_MAP — 핵심 3페이지 모션 지도 (검토 중 작업안, 2026-07-23 v5.1)

> **v5.1 변경 요약**: ①포트폴리오 320vh Pin 제거(실자료 없는 긴 Pin 금지 → 유형 인덱스 축소판) ②제작 방식 Sticky에 예시 산출물 패널 동기화 추가(텍스트 명도 변화만으로 전환 금지) ③X-ray 표면을 실제 페이지형 프레임으로 교체, 모션 사용법 안내 문구 삭제 ④플로팅 CTA 단일 버튼+선택 패널, 하단 640px 구간·폼 입력 중 자동 숨김 ⑤성능: `setInterval` 상시 계산 제거 → scroll 이벤트+rAF 렌더링+IntersectionObserver 활성 구간 게이트, bfcache 복귀(pageshow) 시 상태 강제 마감.

> 모션 유형은 4개로 제한: **A 텍스트 마스크 리빌 / B 클립 리빌 / C Sticky·Pin / D Hover 마이크로**.
> 공통 규칙: 이징 `cubic-bezier(.16,1,.3,1)` · 숨김은 JS가 붙이는 클래스로만 발생(무JS = 전부 표시) ·
> 모든 유형에 안전망 타이머(2.6s) · `prefers-reduced-motion: reduce` 시 A·B·C 비활성(D는 색 전환만 즉시 적용) ·
> 라이브러리 미도입(바닐라 CSS+JS로 충족 — ScrollTrigger 문법은 scroll 리스너+rAF+300ms 인터벌 보조로 재현, 의존성·성능 관리 우선. 부족해지면 도입 전 이 문서에 기록).

## 검수에서 확인한 안전장치
- 초기 로딩 깜박임: 숨김이 JS 클래스 부여 후에만 발생 → JS 미실행 시 처음부터 전부 표시.
- 모션 전 빈 화면: 안전망 2.6s가 미노출 요소 강제 표시(전 유형).
- 성능: 상시 인터벌 계산 없음 — scroll 이벤트에서 상태 갱신, rAF로 렌더링, **IntersectionObserver로 스크럽 대상이 화면 근처에 있을 때만 계산**. transform·clip-path·opacity만 사용(리플로 없음).
- 뒤로 가기(bfcache): `pageshow` 시 숨김 클래스 강제 해제 + 스크럽·플로팅 상태 재계산.
- 검수용 수동 훅: `window.__haoScrub()`.
