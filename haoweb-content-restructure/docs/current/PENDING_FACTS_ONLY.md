# PENDING_FACTS_ONLY — 실제 사실 정보 교체 목록

> 이 문서에는 **임의로 만들 수 없는 사실 정보만** 기록한다.
> 서비스 기획·페이지 콘텐츠·플랜·무료 제안 범위는 모두 확정되어 구현되어 있다(→ `SITE_SPEC_CURRENT.md`).
> 아래 값은 `src/site-config.js` 한 곳에서 교체하면 전 페이지에 반영된다.

## 1. 사업자·법적 정보 (site-config.js › company)
| 키 | 현재 | 필요 값 |
|---|---|---|
| `legalName` | 빈 값 | 사업자등록증상 상호 |
| `ceo` | 빈 값 | 대표자명 |
| `bizNo` | 빈 값 | 사업자등록번호 |
| `address` | 빈 값 | 사업장 주소 |
| `established` | 빈 값 | 설립일(업력 표기용) |

## 2. 연락처 (site-config.js › contact)
| 키 | 필요 값 |
|---|---|
| `tel` | 대표 전화번호 |
| `email` | 대표 이메일 |
| `kakao` | 카카오톡 채널 주소(운영 시) |
| `hours` | 상담 가능 시간 |

## 3. 개인정보 (site-config.js › privacy)
| 키 | 필요 값 |
|---|---|
| `officer` | 개인정보 보호 담당자명 |
| `officerEmail` | 담당자 이메일 |
| `effectiveDate` | 방침 시행일 |
| `analytics` | 분석 도구 사용 여부(GA 등) |

## 4. 폼 수신 (site-config.js › form)
| 키 | 필요 값 |
|---|---|
| `endpoint` | 폼 접수 API 주소 또는 외부 폼 서비스 URL |
| `recipient` | 접수 알림 수신 이메일 |

※ `endpoint`가 비어 있으면 폼은 검증·확인 화면까지만 동작하고 전송하지 않는다(현재 상태).

## 5. 실제 프로젝트 (src/data/portfolio.json)
프로젝트당 필요: 프로젝트명(공개 동의) · 업종 · 제작 유형 · 제작 배경 · 핵심 과제 · 사이트맵/콘텐츠 방향 · 디자인 방향 · 주요 화면(Desktop) · Mobile 화면 · 제작 범위 · 관련 서비스
※ 현재 `[]` — 데이터가 없으면 공개 dist에서 포트폴리오 목록·상세가 빌드되지 않는다.

## 6. 실제 고객 인터뷰 (src/data/interview.json)
인터뷰당 필요: 고객사·담당자(공개 동의) · 제작/리뉴얼 이유 · 기존 문제 · 업체 선택 기준 · 하오웹 선택 이유 · 진행 과정 경험 · 오픈 이후 활용 · 조언
※ 현재 `[]`

## 7. 팀·협력
| 항목 | 필요 값 |
|---|---|
| 공개 가능한 팀 소개 | 실명·역할·경력 (공개 여부 포함) |
| 협력 전문가 | 번역·감수·촬영 등 실제 협력 구조 |
| 해외 사무실 (확정) | **하오웹은 베트남·중국·태국에 현지 사무실을 운영한다**(2026-07-24 실무 확인). global 및 global-vn/cn/th에 신뢰 요소로 반영 — 표현은 "현지 사무실 운영 + 국내 제작과 현지 커뮤니케이션 연결"까지만. |
| 해외 사무실 상세 (미입력) | 각 사무실의 정확한 **주소·전화번호·담당자명·직원 인원**은 확인 시에만 기재. |
| 미확인 — 만들지 않음 | 다음은 확정 사실이 아니므로 서술 금지: 현지 직원이 번역/원어민 감수/국가별 검색 환경 분석/현지 법률 검토 수행, 현지 사무실의 전문 SEO 수행, **일본·영어권 협력 전문가 존재**, 국가별 전문 번역 인력 상주. 번역·감수는 "국가·콘텐츠 범위에 따라 진행 방식 구성"으로만 표현. |

## 8. 실적·수치
가상의 프로젝트 수·고객 수·성과 수치·후기·검색 순위 변화는 **어떤 경우에도 생성하지 않는다.**
실제 수치가 확보되면 portfolio.json과 about에 반영한다.

---

### 공개 배포 전 필수 입력
1. `company` 전체 · `contact.tel` · `contact.email`
2. `privacy.officer` · `privacy.effectiveDate` → 입력 전까지 privacy.html은 공개 Footer 미연결(review 전용)
3. `form.endpoint` → 입력 전까지 폼은 전송하지 않음
4. `portfolio.json` 1건 이상 → 입력 전까지 포트폴리오는 dist 제외
