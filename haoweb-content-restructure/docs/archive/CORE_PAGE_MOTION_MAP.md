> 실제 구현과 일치하는 내용만 기록한다. 이전 단계 기록은 `archive/CORE_PAGE_CHANGES_HISTORY_v5.md`.
> 모션 유형 4개로 제한: **A 텍스트 마스크 리빌 / B 클립 리빌 / C Sticky / D Hover·Focus 마이크로**.
> 공통 규칙: 이징 `cubic-bezier(.16,1,.3,1)` · 숨김은 JS가 붙이는 클래스로만 발생(무JS = 처음부터 전부 표시) ·
> 안전망 타이머 2.6s · `prefers-reduced-motion: reduce` 시 A·B·C 비활성(D는 색 전환만 즉시) ·
> **라이브러리 미도입** — ScrollTrigger 문법은 scroll 이벤트 + rAF 렌더링 + IntersectionObserver 활성 구간 게이트로 재현(의존성·성능 관리 우선). 도입 시 이 문서에 사용 위치·이유·Mobile 대체·reduced-motion·성능 영향을 기록한다.
> **모션보다 시각 콘텐츠 우선**: 이번 단계에서 모션 수를 늘리지 않고, 전환할 실제 자료(작업 문서·구조도·화면 프레임)를 먼저 확보했다.

## 안전장치
- 초기 로딩 깜박임: 숨김은 JS 클래스 부여 후에만 발생 → JS 미실행 시 처음부터 전부 표시.
- 모션 전 빈 화면: 안전망 2.6s가 미노출 요소 강제 표시.
- 성능: 상시 인터벌 없음 — scroll 이벤트 갱신 + rAF 렌더링 + IO 활성 구간 게이트. transform·clip-path·opacity만 사용(리플로 없음).
- 뒤로 가기(bfcache): `pageshow`에서 숨김 클래스 강제 해제 + 스크럽·플로팅 상태 재계산.
- 검수용 수동 훅: `window.__haoScrub()`.
