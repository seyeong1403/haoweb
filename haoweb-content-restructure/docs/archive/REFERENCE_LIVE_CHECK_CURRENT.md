# REFERENCE_LIVE_CHECK_CURRENT — 레퍼런스 실사이트 재확인 (2026-07-23 라이브 실측)

> 문서 의존 금지 원칙에 따라 작업 직전 실사이트를 Desktop·Mobile로 재확인한 기록.
> 문구·이미지·고객사·소스코드 복사 금지 — 원리만 하오웹 섹션에 연결한다.

## 실측 요약

### 잇다소프트 itddaa.com (주)
- Desktop: 고정 투명 헤더 40px(스크롤 후에도 유지·과한 숨김 없음), 히어로 100vh 점유, 디스플레이 95~105px(-4% 자간), 섹션 여백 0~125px로 **장면마다 밀도 차이가 큼**, 상시 앰비언트 루프(3~5s 플로팅·컬러 페이드), 필 버튼 radius 5px·소형
- Mobile 375px: 디스플레이 72px(≈19vw)로 **타이포 스케일 유지**, 애니메이션 39개 유지(제거보다 감량), 메뉴 버튼만 노출
- 핵심: 큰 타이포 옆에 **실물(오브젝트 사진·주석 라인)**이 항상 함께 있음 — 타이포만으로 화면을 채우지 않음

### 도루코 dorco.co.kr (보조)
- Pretendard Std, 헤딩 자간 -1.6~-3%, 12px/500 소형 라벨, 섹션 여백 69~139px
- 레드는 라인·소형 포인트·상태 변화로만 — 대면적 금지
- 화이트 기조 + 순블랙 섹션 교차, 풀스크린 영상은 하오웹 미도입(자산 없음)

### 파이브프로 fivepro.co.kr (콘텐츠·프로젝트 전개)
- 헤더 85px 비고정, 히어로 60px(모바일에서도 60px 유지)
- **프로젝트 이미지는 세로로 긴 페이지 전체 캡처(비율 0.32~0.38)** — 실제 구축 화면이 신뢰의 근거
- 인용형 "고객사 | 한 문장" 전개, 표 다수(14개) — 비교·범위 표기 문법
- 모바일에서도 애니메이션 다수 유지(88), 정보 누락 없음

## 원리 → 하오웹 섹션 연결

| 레퍼런스 원리 | 하오웹 적용 위치 |
|---|---|
| 잇다: 타이포 진입(줄 마스크) + 옆에 실물 배치 | 메인 Hero — H1 유지 + **작업 보드(하오웹 제작 서식 겹침)** 를 우측에 배치, 빈 공백 해소 |
| 잇다: Sticky 장면에서 실제 내용이 전환 | 제작 방식 — 좌측 고정 보드의 **작업 문서가 단계와 함께 실제 교체** |
| 잇다: 장면별 밀도 차이(0~125px) | 메인 섹션 여백 차등(마퀴 제거, 조밀 장면·여백 장면 교차) |
| 도루코: 선·레드 상태 변화 | Header GNB 밑줄 성장, 서비스 인덱스 행 반전, 소형 레드 라벨은 red-dark로 대비 확보 |
| 도루코: 12px 라벨 문법 | 작업 문서 헤더·플로우 라벨 |
| 파이브프로: 실제 구축 화면(세로 캡처) 중심 프로젝트 전개 | **실제 프로젝트 확보 후** portfolio A안(화면 점유 쇼케이스)의 이미지 규격 기준(세로 전체 캡처+Desktop·Mobile 병행) |
| 파이브프로: 표 기반 범위 비교 | website 유형 비교표(Desktop) — Mobile은 세로 블록 전환 |
| 파이브프로: 모바일 정보 무손실 | 제작 방식·유형 미리보기 — Mobile에서 6개 산출물·7개 구조도 전부 노출 |

## 마퀴 제거 판단
잇다의 마퀴는 포트폴리오·실물 콘텐츠 사이의 리듬 장치로 기능하지만, 하오웹 메인에서는 Header·서비스 섹션 반복+중심축 희석이라 제거(지시 §3와 일치). 전환은 여백·배경 전환으로 처리.
