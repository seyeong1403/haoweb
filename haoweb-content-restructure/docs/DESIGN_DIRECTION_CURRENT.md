## 1. 레퍼런스 역할

## 2. 하오웹 고유 원칙
- 가독성과 신뢰가 우선. 편집형 연출은 이를 해치지 않는 범위에서.
- 실제 회사 정보·성과·고객사·수치 임의 생성 금지.
- 개발 상태를 고객에게 설명하는 공개 문구 금지(HTML 주석·내부 문서로만).
- AI는 수단으로 서술, Hero 중심 메시지 금지.

## 4. 컬러 사용 비율
- White 바탕 약 75~80% / near-black(#0A0A0B) 다크 장면 15~20% / **HAOWEB Red(#E83817) 5% 이하**.
- 레드 허용: h1 위 4px 바(도루코 라인), 마침표·강조 단어, 번호, 활성/hover 상태, primary 버튼.
- 레드 금지: 대면적 배경, 모든 제목, 모든 버튼(ghost는 잉크 아웃라인).

## 5. 레이아웃 규칙
- wrap 1200px. 구획은 카드가 아니라 **선(헤어라인·2px 탑룰)과 여백**.
- 라운드·그림자 남용 금지(라운드는 버튼 필·폼 요소만).
- 화면마다 타이포·이미지·여백 비율에 변화를 줄 것 — 같은 섹션 구조 반복 금지.
- 비대칭 그리드(주 항목 크게, 보조 작게).
- 섹션 여백 데스크톱 110~130px, 풀블리드 장면(split·핀)은 wrap 밖 사용 가능.

## 6. 모션 규칙 (4유형 — CORE_PAGE_MOTION_MAP.md에 장면별 상세)
- A 텍스트 마스크 리빌: 줄 단위 overflow mask + 상승. Hero h1·핵심 h2만.
- B 클립 리빌: 화면·이미지 블록. clip-path inset, 텍스트보다 늦게.
- C Sticky·Pin: 핵심 장면 1~2곳만(제작 방식, 포트폴리오).
- D Hover 마이크로: 선 길이·화살표 이동·색 반전·미세 이동. **lift(떠오름) 금지**.
- 이징 cubic-bezier(.16,1,.3,1), duration 0.7~1s, 스태거 30~110ms.
- 전 모션: JS 게이트(.js-anim — 무JS는 전부 표시), 안전망 타이머, prefers-reduced-motion 시 비활성.
- 라이브러리 미도입(바닐라 CSS+JS로 충족, 성능·의존성 관리 우선). 필요해지면 도입 전 기록.

## 7. Mobile 규칙
- 핀·스크럽 해제 → 일반 흐름. 마퀴는 유지(저비용 CSS).
- 디스플레이 타이포 크기 유지(축소 최소화), 여백 64px±.
- hover 의존 정보 금지(설명 상시 노출), 터치 영역 44px+.
